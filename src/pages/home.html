// Create a query to find some trees in the DB
$fQuery = Tree::faceQueryBuilder()
    ->join("lemons") // trees have lemons
    ->join("lemons.seeds") // each lemon has some seeds
    ->join("leafs")  // trees have also leafs
    ->where("~age>10"); // only tree older than 10



// create a PDO object :
$pdo = new PDO('mysql:host=localhost;dbname=lemon-trees', 'login', 'pswd');

// Get the datas from the db
$trees =  Face\ORM::execute($fQuery, $pdo);


echo count($trees);
echo count($trees[0]->getLeafs());
echo $trees[0]->getLeafs()[0]->getSize();

Functional

Face is feature rich. It's not only a query builder, it is first of all a powerfull abstraction of your database structure.

Performant

The library was written with performances in mind. It uses as few layers as possible and you can adapt the queries to get the best performances

Time saver

Dealing with complex queries becomes very easy and transparent. With a very few efforts you can write join queries and get a structured result. The workflow is very transparent and semantic

Easy Integration

Face is easy to integrate and won't be in conflict with you existing code. Models classes use traits instead of inheritance.
